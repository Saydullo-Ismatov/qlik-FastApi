Stock App — Roadmap

Tables: stock_qty & application_status

Start: 22 Apr 2026 Target completion: 13 May 2026 (≈ 3 weeks) Ownership: Qlik → Plan Dept (customer) → Savdo (testing)

DONE What has been completed

  • Added export_native endpoints for stock_qty and application_status tables Supports excel / csv / tsv / parquet via Qlik's native ExportData
  • Fixed GetLayout API error WebSocket recv timeout increased (5× base) for large layouts
  • Optimized stock app pagination Session hypercube reuse, faster paging on large datasets
  • Fixed SelectValues numeric field handling Session hypercube filtering now works for numeric dimensions
  • Stock app blank-measure bug fixed Field selection issues that produced empty rows resolved
  • Parquet fallback logic Auto-fallback to Excel on Qlik error 3004

TO DO What still needs to be done

  • Refactor project structure — isolate apps Each app (stock / factory_data / material_remainder / application_status) must live in its own module so they no longer interfere with one another. Shared code into a common layer; per-app endpoints, services, repos decoupled.
  • Implement export_native with ALL filters Filters are currently not applied to native Qlik export. Must honour selections (field filters, multi-select), variables (vChooseType etc.), and bookmarks — for both stock_qty and application_status.
  • Internal data-correctness testing (Qlik team) Verify exported data is complete and identical to what Qlik Sense itself returns under the same filters. Row counts, totals, edge filters, empty selections.
  • Hand-off to Savdo for acceptance testing Only after internal verification passes. Savdo runs the test conditions (data match, functional correctness, refresh cadence).
  • UAT + project close-out Plan Department demo & sign-off → Принятие Гранта.

Acceptance criteria

  • Per-app isolationChanges in one app's code (e.g. stock_qty) must not break other apps. Verified by running regression on factory_data and material_remainder after refactor.
  • All filters work on export_nativeEvery filter supported on the data endpoints (selections, variables, bookmarks) must produce identical filtered output when used with export_native.
  • Data parityRow counts, totals and filter behaviour must match Qlik Sense UI output for both stock_qty and application_status.
  • Savdo sign-offSavdo confirms: соответствие ТЗ/ФС, соответствие данных, работа функционала, частота обновления.
  • Plan Dept UAT acceptedFinal demo + training → Принятие Гранта.
Risks / watch-out: Qlik native export does not always honour session hypercube selections the same way the data endpoints do — expect the filter implementation for export_native to require re-applying selections through the Engine API before calling ExportData. Parquet error 3004 already has a fallback; keep it in the refactored structure.
